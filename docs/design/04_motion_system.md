# Motion System

## モーションの前提

アニメーションは「何かが動いている」こと自体が目的ではありません。
役割は次の3つです。

- 情報の優先順位を見せる
- ページ体験に記憶を残す
- 技術力や作り込みを感じさせる

## まず守るべき原則

### 1. 読めることを優先する

動きが強くても、文言が読みにくいなら失敗です。

### 2. 1ページに複数の動き方を入れすぎない

パララックス、sticky、scale、blur、morph を全部同時に入れると雑になります。

### 3. 止まる理由があるときだけ固定する

sticky や pinned 表現は強いですが、
使いどころを間違えると「壊れている」に見えます。

## 推奨するモーション

### Hero Drift

背景、グリッド、プレビューカードがそれぞれ少し違う速度で動く。

狙い:

- 奥行きを出す
- ただの静的ヒーローを避ける

### Soft Reveal

セクションが下からふわっと出るのではなく、
少しだけスライドしながら静かに現れる。

狙い:

- 安っぽさを避ける
- 高級感を保つ

### Layered Card Motion

カードごとにわずかに違うY方向の移動を与える。

狙い:

- 同じ3カラムでも単調さを減らす
- 空間感を足す

### Controlled Sticky

どうしても使うなら、1ページに1箇所まで。

向いている場所:

- プロセス説明
- ケーススタディ比較
- 数値や実績の切り替え

条件:

- 固定の理由が明確
- モバイルでは簡略化する
- 戻る操作で破綻しない

## 避けるべきモーション

### 理由のない常時アニメーション

常に何かが脈動している、光っている、回っている状態は避ける。

### 過剰なスクロールジャック

ページが思ったように進まない体験は、よほど強いコンセプトがない限り危険です。

### 派手だが意味のない 3D

視差や回転は少量なら効きますが、
常に強い3Dがかかると UI の信頼感を落とします。

## 実装判断の基準

モーションを入れる前に、毎回次を確認する。

- これは情報理解を助けるか
- これはブランド印象を上げるか
- これがなくても成立する設計になっているか

この3つのうち、2つ以上に yes と言えない動きは削るべきです。
